# 成熟开源量化机器人对比

如果你担心当前项目不够成熟，可以考虑下面几个 GitHub 项目。

## 1. Freqtrade

结论：**最推荐作为长期底座。**

适合：

- Python 用户
- 接交易所 API
- 回测
- dry-run
- 实盘
- Web UI
- 自定义策略

优点：

- 社区成熟
- 文档完整
- 支持很多交易所
- 有 dry-run 和实盘模式
- 风控、策略、回测、日志都比较系统

缺点：

- 配置还是复杂
- 第一次启动不算无脑
- 需要我帮你包一层中文一键脚本

## 2. OctoBot

结论：**界面友好，但策略/工程可控性不如 Freqtrade。**

适合：

- 不想写代码
- 更偏可视化操作
- 想先点点界面熟悉

缺点：

- 定制研究策略可能不如 Freqtrade 顺手

## 3. Hummingbot

结论：**专业但复杂，不建议你第一步用。**

适合：

- 做市
- 网格
- 套利
- 多交易所
- 专业 crypto market making

缺点：

- 学习成本高
- 不是最适合“无脑先用起来”

## 4. Jesse

结论：**适合研究和写策略，不是最无脑。**

适合：

- 策略开发
- 回测
- Python 研究

缺点：

- 更像框架，不像现成 app

## 我的建议

现在有两条路：

1. **继续用当前项目**：我已经把它封装成中文双击版，最适合你先接真实账号但不存钱。
2. **换 Freqtrade 做底座**：更成熟，但我需要再花一轮把 Freqtrade 包成中文一键启动版本。

如果你想认真长期用，我建议下一阶段我帮你搭一个：

```text
freqtrade-cn-oneclick/
  1_安装.command
  2_编辑交易所配置.command
  3_启动WebUI.command
  4_开启dry-run.command
  5_小资金live前检查.command
```

这样成熟底座归 Freqtrade，中文无脑入口归我们自己封装。
