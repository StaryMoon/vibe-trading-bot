# 量化机器人中文傻瓜版说明

你现在只需要记住一句话：

**先接真实交易所账号，但可以先不存钱。没 USDT 的时候，真实买单不会成交。**

## 你要点哪个文件

第一次用：

1. 双击 `1_编辑API配置.command`
2. 在打开的 `.env` 里填 Binance API key
3. 双击 `2_检查真实交易配置.command`
4. 双击 `3_打开真实交易控制台.command`

出问题或不想让它交易：

- 双击 `4_紧急停止.command`

恢复：

- 双击 `5_解除紧急停止.command`

关闭网页控制台：

- 双击 `6_停止控制台.command`

只想 5 USDT 试一下：

- 双击 `7_打开5U真实试用控制台.command`

## .env 里最少要填什么

```text
VIBE_TRADER_MODE=live
BINANCE_API_KEY=这里填你的key
BINANCE_SECRET=这里填你的secret
LIVE_TRADING_ACK=true
LIVE_TRADING_CONFIRM_TEXT=I_UNDERSTAND_REAL_MONEY_RISK
MAX_LIVE_EQUITY=50
```

`MAX_LIVE_EQUITY=50` 的意思是：这个机器人最多只允许你暴露 50 USDT 级别的资金。

如果你只想先放 5 USDT，把它改成：

```text
MAX_LIVE_EQUITY=5
```

然后用 `7_打开5U真实试用控制台.command`。

## 5 美元怎么存

机器人不能帮你存钱，存钱必须在交易所 App/网页里完成。

最简单的路径：

1. 打开 Binance 或你实际使用的交易所 App。
2. 进入 `钱包 / Wallet`。
3. 选择 `充值 / Deposit` 或 `买币 / Buy Crypto`。
4. 如果支持银行卡/ACH/Apple Pay，就直接买或充值大约 5 USDT。
5. 确认钱在 `现货 / Spot` 账户里，不要放在理财、资金账户或合约账户。
6. 回到机器人控制台，点 `刷新一轮`，看余额是不是显示出来。

如果你是从另一个钱包转 USDT：

1. 交易所里点 `充值 / Deposit`。
2. 选择 `USDT`。
3. 选择网络时必须和转出钱包完全一致。
4. 5 USDT 很小，网络手续费可能不划算；不要用手续费很贵的网络试小额。
5. 先小额测试，确认到账后再动机器人。

注意：5 USDT 可能刚好卡在交易所最小下单额附近。如果下单失败，不一定是程序坏了，可能只是交易所最小订单或手续费限制。更稳的是先放 10-20 USDT，但你坚持 5U 也可以先测试连接和余额。

## Binance API key 怎么开

只开：

- 读取
- 现货交易

不要开：

- 提现
- 合约
- 杠杆
- 资金划转

## 可以先不存钱吗

可以。

没有 USDT 时：

- 可以打开真实交易控制台
- 可以检查 API 是否能连接
- 可以看真实账户余额
- 但买单不会成交

等你真的要小资金试用，再往现货账户里放很少的 USDT，比如 20-50 USDT。

## 网页控制台里按钮是什么意思

- `刷新一轮`：读取行情和账户，跑一遍策略和风控。
- `暂停`：停止交易动作。
- `恢复`：允许交易动作。
- `紧急停止`：强制拦截所有交易动作。
- `解除紧急停止`：恢复前的必要步骤。
- `提交手动单`：手动买入或卖出，仍然会先过风控。

真实实盘模式下，提交手动单前还要输入：

```text
EXECUTE_REAL_ORDER
```

## 我建议你第一次怎么做

1. 先不要存钱。
2. 填好 API key。
3. 跑 `2_检查真实交易配置.command`。
4. 打开控制台，看余额是否是 0。
5. 确认能正常显示以后，再考虑只存 20-50 USDT。

## 成熟开源项目怎么选

我查了一圈，成熟项目里最值得考虑的是：

- **Freqtrade**：最适合作为长期底座，Python、支持 WebUI、回测、dry-run、实盘、交易所接入成熟。
- **OctoBot**：界面更友好，适合不想写代码的人。
- **Hummingbot**：更偏做市和专业交易框架，功能强但复杂。
- **Jesse**：适合写策略和研究，不是最无脑。

如果后面要换成熟底座，我建议优先试 **Freqtrade**。但它也有配置复杂度，所以我会负责把它封装成你能双击启动的版本。
